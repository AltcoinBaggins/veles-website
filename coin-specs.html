General specifications :
  • Coin supply : 12 500 000 VLS
  • Block time : 2 minutes
  • Swap premine : 50000 VLS
  • Masternode collateral : 2000 VLS
  • Multi-algorithmic : Lyra2z, X11, X16R, Sha256d, Scrypt, Nist5
  • Active and only dev. address : VKtig9w7wfx6Ep9B4NnbwxLj1bkY2kHXBj

    • Reward system :
  • Minimal block reward : 0.01 VLS
  • Maximal Block reward : 80 VLS
  • Proof of work reward : Rewards are dynamicaly moving between Max. and Min. rewards . The movement of rewards is connected with hashrate (more hashrate = bigger reward, less hashrate = lower reward)
  • Masternode reward : Dynamicaly raising from 5% to 60% of block reward in 5 years.
  • Development reward : Dynamicaly decreasing from 5% to 0% in 5 years.

    • Halving system :
  • Veles has own custom dynamic nonlinear halvings. Here is functionality and usage explained.You can check Veles halving status with rpc command "gethalvingstatus".

  • Example of gethalvingstatus :
    {
    "halvings_occured": 1,
    "epochs_occured": 5,
    "halving_interval": 4000,
    "blocks_to_next_epoch": 1579,
    "epoch_supply_target_reached": "14%",
    "min_epoch_supply_to_halve": "80%",
    }
  • It starts with the most basic information, this one here says, that the next block reward halving is supposed to occur in 1579 blocks from now, but most likely it won't occur, as we've mined only 14% from required 80% of maximum theoretical block reward since the last halving (because of the dynamic block reward that depends on the hashrate - number of workers). And If the halving doesn't occur, it will be delayed another 4000 blocks. But the dynamic block rewards will actually increase (instead of being halved), so that we'll reach our target (of coins to be mined till next halving) sooner and won't be delaying the halving unnecessarily for too long, even when the hashrate is lower than ideal.   • Here we can see, for example, that at the block 52k halving has not occurred because there were only 51% of the target amount of VLS mined, from required 80%. The maximum block reward was 80 VLS.
  • So the halving was delayed by another 2k blocks and this constitutes another halving epoch (APLHA_H0_E1), where the max_block_reward remains 80 VLS, but the dynamic_rewards_boost kicks in and increases dynamic blocks rewards by 50% (but always within the range of maximum block reward of 80 VLS). Thanks to this mechanism we can see that at the and of this epoch the initial target was actually fulfilled by 125% (as the time has increased too, those extra % are just fine)
