Guía dVPN para Linux

Instalando OpenVPN

Si estás usando Linux, hay una gran variedad de herramientas que puedes utilizar, dependiendo de tu distribución. Tu entorno de escritorio o administrador de conexiones pueden también incluir utilidades de conexión. La manera mas universal de conectarse, de todas formas, es utilizar el software OpenVPN.

En Ubuntu o Debian, puedes instalarlo de la misma forma que en el servidor ingresando:

  • sudo apt-get update
  • sudo apt-get install openvpn

En CentOS puedes habilitar los repositorios EPEL y luego instalarlo ingresando:

  • sudo yum install epel-release
  • sudo yum install openvpn


Configurando OpenVPN

Descarga los archivos de configuración de Veles y muévelos a su repositorio ingresando:

  • wget https://explorer.veles.network/dapi/getOpenVPNConfig
  • sudo mv veles.ovpn /etc/openvpn/client/


Para permanecer protegido contra leaks de DNS usa el siguiente comando (IP utilizada en este caso sirve solamente como ejemplo y debe ser reemplazada por una de la nueva lista de Masternodes )

  • echo -e "#Veles decentralized DNS\nnameserver 111.111.111.111" | sudo tee -a /etc/resolv.conf >> /dev/null

Usuarios de interfaz gráfica deben seguir los siguientes pasos.
Primeramente clickear con el botón derecho en el ícono del administrador de conexiones de red y clickear en "Editar Conexiones... (Edit Connections...)" como se muestra a continuación:

Ahora debemos presionar "+" para agregar una nueva conexión

Elegimos la opción "Importar un archivo de configuración guardado" ("Import a saved configuration file..." para importar nuestra configuración

Ahora, ir a la ubicación de la configuración, en nuestro caso será /etc/openvpn/client/ y elegir veles.ovpn

No debes alterar nada, se creará automáticamente el perfil VPN de Veles y sólo debes presionar "Guardar" ("Save")



Conectando el cliente OpenVPN

Usuarios de CLI deben seguir estos pasos.
Ahora, puedes conectarte a la VPN simplemente apuntando el comando al archivo de configuración del cliente:

  • sudo openvpn --config /etc/openvpn/client/veles.ovpn

Usuarios de interfaz gráfica deben seguir los siguientes pasos.
Abrir el Administrador de Conexiones de Red haciendo click derecho en su ícono. Allí verás la sección "Conexiones VPN"("VPN Connections"). Click en ella. Marca "veles" y disfrutá tu conexión!

Instalando Stunnel

Aquí explicamos el proceso de instalación del cliente Stunnel como cliente en Linux.

En Ubuntu o Debian, puedes instalarlo ingresando:

  • sudo apt-get install stunnel

En CentOS puedes instalar Stunnel ingresando:

  • yum install stunnel


Configurando Stunnel

Descarga los archivos de configuración de Veles para Stunnel y mueva sus repositorios ingresando:

  • wget https://explorer.veles.network/dapi/getStunnelConfig
  • wget https://explorer.veles.network/dapi/getStunnelCertificate
  • wget https://explorer.veles.network/dapi/getShieldedOpenVPNConfig
  • sudo mv shielded-veles.ovpn /etc/openvpn/client/
  • sudo mv veles.stunnel.conf /etc/stunnel/stunnel.conf
  • sudo mv stunnel.pem /etc/stunnel/


Concetando el cliente OpenVPN con Stunnel

Usuarios de CLI deben seguir los siguientes pasos.
Puedes conectarte a la VPN a través de Stunnel simplemente encendiendo Stunnel y apuntando el comando openVPN al archivo de configuración del cliente:

  • sudo stunnel
  • sudo openvpn --config /etc/openvpn/client/veles-shield.ovpn

Usuarios de interfaz gráfica deben seguir los siguientes pasos.
Sigue los pasos de la guía OpenVPN e importa veles-shield.ovpn y no olvides encender Stunnel

Instalando el proxy de Ofuscación

Aquí explicamos el proceso de instalar ObfsProxy como cliente en Linux.

En Ubuntu o Debian, puedes instalarlo ingresando:

  • sudo apt-get install obfsproxy


Configurando

Descarga los archivos de configuraciín de Veles y muevelos a sus repositorios ingresando:

  • wget https://explorer.veles.network/dapi/getShieldedOpenVPNConfig
  • sudo mv shielded-veles.ovpn /etc/openvpn/client/


Conecctando el cliente OpenVPN con ObfsProxy

Usuarios de CLI deben seguir los siguientes pasos.
Conecta a la VPN iniciando ObfsProxy y apuntando el comando OpenVPN Al archivo de configuración del cliente:

  • sudo obfsproxy scramblesuit --dest 80.211.132.243:21343 --password=SSSTIZ3LG5SSS43OMSSSM4LKGQFASSSS client 127.0.0.1:21337
  • sudo openvpn --config /etc/openvpn/client/veles-shield.ovpn


La IP de destino (* en este caso --dest 80.211.132.243) debe ser reemplazada con la IP del Masternode al que te estás conectando. Usaremos este password sólo en la etapa de testeo pre-alpha. En el futuro en lugar del password se utilizará un string aleatorio.


Usuarios de interfaz gráfica deben seguir los siguientes pasos.
Siga los pasos de la guía de OpenVPN e importe veles-shield.ovpn sin olvidar iniciar ObfsProxy

Testeá tu conexión VPN

Una vez que todo se encuentra instalado, un simple chequeo confirma que todo funciona correectamente. Sin tener la conexión VPN activada, abre el navegador y ve a DNSLeakTest.
El sitio retornará la dirección IP que se te ha sido asignada por tu proveedor de internet y cómo apareces antes el resto del mundo. Para chequear tu configuración DNS a través del mismo sitio, clickea en Extended Test y te informará que servidores DNS estás usando.
Ahora, conecta tu cliente VPN y refresca el navegador. Debería aparecer la IP completamente nueva de tu servidor VPN. Así es como aparecerás al mundo. Nuevamente, el Extended Test de DNSLeakTest chequeará tu configuración DNS y confirmará que ahora estás usando los servidores DNS publicados por la dVPN de Veles.
