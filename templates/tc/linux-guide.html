Linux dVPN 教程

安裝OpenVPN

如果您使用的是Linux，根據您的發行版，可以使用多種工具。 當然，最普遍的連接方式是使用OpenVPN軟體。

在Ubuntu或Debian上，您可以使用這樣的命令

  • sudo apt-get update
  • sudo apt-get install openvpn

在CentOS上，您可以啟用EPEL存儲庫，然後通過以下命令安裝：

  • sudo yum install epel-release
  • sudo yum install openvpn


配置OpenVPN

下載Veles配置檔並輸入以下內容移動到其存儲庫：

  • wget https://explorer.veles.network/dapi/getOpenVPNConfig
  • sudo mv /etc/openvpn/client/veles.ovpn


防止DNS洩漏，請使用此命令（本例中使用的IP僅作為示例，需要替換為新主節點列表中的IP）

  • echo -e "#Veles decentralized DNS\nnameserver 111.111.111.111"

對於GUI用戶，請遵循以下步驟。
首先用滑鼠右鍵單擊網路管理器圖示，然後單擊編輯連接

現在，我們將點擊“+”符號添加新連接

我們將選擇“導入保存的配置檔...”導入我們的配置

現在找到我們的配置位置，在我們的例子中，它是在/etc/openvpn/client/並選擇veles.ovpn

您不需要幹其他事情了，它會自動為您創建Veles VPN配置檔，您只需按“保存”



連接OpenVPN

對於CLI用戶，請遵循以下步驟。
現在，只需將openvpn指向配置檔，就可以連接到VPN：

  • sudo openvpn --config /etc/openvpn/client/veles.ovpn

對於GUI用戶，請遵循以下步驟。
用滑鼠右鍵單擊網路管理器的圖示打開它。在這裏您將看到“VPN連接”。點擊它。標記“veles”並享受您的連接！

安裝 Stunnel

我會介紹怎麼在Linux中將Stunnel作為客戶端安裝的過程。

在Ubuntu或Debian上，輸入如下命令：

  • sudo apt-get install stunnel

在CentOS上，輸入如下命令：

  • yum install stunnel


配置 Stunnel

下載Veles Stunnel配置檔，通過如下命令移動到其存儲庫：

  • wget https://explorer.veles.network/dapi/getStunnelConfig
  • wget https://explorer.veles.network/dapi/getStunnelCertificate
  • wget https://explorer.veles.network/dapi/getShieldedOpenVPNConfig
  • sudo mv shielded-veles.ovpn /etc/openvpn/client/
  • sudo mv veles.stunnel.conf /etc/stunnel/stunnel.conf
  • sudo mv stunnel.pem /etc/stunnel/


用Stunnel連接OpenVPN客戶端

對於命令行用戶使用如下步驟
現在，只需打開Stunnel並將OpenVPN命令指向客戶端配置檔，就可以通過Stunnel連接到VPN：

  • sudo stunnel
  • sudo openvpn --config /etc/openvpn/client/veles-shield.ovpn

對於圖形化介面的用戶使用如下步驟
按照OpenVPN指南中的步驟導入veles-shield.ovpn，不要忘記打開Stunnel

安裝 Obfuscation Proxy

我會介紹怎麼在Linux中將ObfsProxy作為客戶端安裝的過程。

在Ubuntu或Debian上，輸入如下命令：

  • sudo apt-get install obfsproxy


配置

下載Veles Stunnel配置檔，通過如下命令移動到其存儲庫：

  • wget https://explorer.veles.network/dapi/getShieldedOpenVPNConfig
  • sudo mv shielded-veles.ovpn /etc/openvpn/client/


Connecting OpenVPN Client with ObfsProxy

對於命令行用戶使用如下步驟
現在，只需打開ObfsProxy並將OpenVPN命令指向客戶端配置檔，就可以連接到VPN：

  • sudo obfsproxy scramblesuit --dest 80.211.132.243:21343 --password=SSSTIZ3LG5SSS43OMSSSM4LKGQFASSSS client 127.0.0.1:21337
  • sudo openvpn --config /etc/openvpn/client/veles-shield.ovpn


目標IP地址（* 在這種情況下為--dest 80.211.132.243）應該替換為您要連接的主節點的IP。我們僅在Alpha測試前階段使用此密碼。 未來會使用隨機字串代替密碼。


對於圖形化介面的用戶使用如下步驟
按照OpenVPN指南中的步驟導入veles-shield.ovpn並不要忘記打開ObfsProxy

測試您的VPN連接

一旦所有的東西都安裝好了，只需要一個簡單的檢查就可以確認他是否運作。在未啟用VPN連接的情況下，打開流覽器並轉到 DNSLeakTest。
該站點將返回由您的網路服務提供商分配的IP地址，以及您在世界其他地方顯示的IP地址。若要通過同一網站檢查您的DNS設置，請單擊擴展測試（Extended Test），它會告訴您正在使用的DNS伺服器。
現在，連接到您的VPN客戶端並刷新流覽器，應該會顯示與您的VPN伺服器的完全不同的IP地址。然後再次進行擴展測試（Extended Test）檢查您的DNS設置，並確認您正在使用Veles dVPN推送的DNS解析。
